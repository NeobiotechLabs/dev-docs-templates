# Safe Chunk Writer Skill

## 🚨 문제 상황 (왜 이 스킬을 써야 하나요?)
당신(AI)이 작성해야 할 산출물 내용이 매우 길거나, 내부에 특수기호(따옴표 등)가 많을 때 `text_editor`나 `shell` 도구를 한 번에 호출하려고 하면 JSON 구조가 변질되어 심각한 파싱 에러(`missing field 'command'`)가 발생합니다.

## 💡 해결 로직: "Chunking & Append"
긴 텍스트를 파일에 쓸 때는 **반드시 15~20줄 단위로 잘게 쪼개서(Chunking)** 파일 뒤에 이어 붙이는(Append) 방식을 사용해야 합니다.

### 📝 실행 절차
1. 변경할/생성할 파일이 이미 있다면 `rm filename` 또는 `echo "" > filename` 으로 초기화하세요.
2. `shell` 도구를 사용하여 터미널의 기본 기능인 `echo -e "..." >> filename` 명령어 패턴을 사용합니다.
3. 한 번 `shell` 도구를 호출할 때 전달하는 텍스트 문자열(Chunk)은 **대략 50줄 내외**로 구성하여 속도 저하를 방지하세요.
4. 첫 번째 청크 쓰기가 성공하면, 다음 청크를 동일하게 `echo -e "다음 내용..." >> filename`으로 이어 붙입니다.
5. 문서 전체가 완성될 때까지 이 과정을 여러 번의 `shell` 통신 턴으로 반복합니다.

### ❌ 절대 금지 사항
- 한 번의 도구 호출에 **80줄 이상**의 거대한 텍스트를 파라미터로 밀어넣지 마세요. (GLM 모델의 JSON 이스케이프 파싱이 깨질 확률이 급증합니다.)
- 파이썬 `-c` 플래그를 활용한 거대 리스트(`lines = ['줄1', ...]`) 방식을 사용하지 마세요. (여전히 따옴표 이스케이프 버그가 발생합니다.)
- bash의 heredoc (`cat << 'EOF'`) 방식을 절대 사용하지 마세요. (예기치 않은 시스템 변수 치환 버그가 발생합니다.)
